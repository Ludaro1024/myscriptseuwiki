**myBilling** ist ein erweitertes Abrechnungssystem Es bietet eine Vielzahl von Funktionen, die über die Standard-Abrechnungssysteme hinausgehen.

## Hauptfunktionen von myBilling

- **Rechnungsbestätigung**: Bevor eine Rechnung wirksam wird, muss der Empfänger sie akzeptieren. Dies verhindert das unautorisierte Versenden von Rechnungen.
- **Teilzahlungen**: Offene Rechnungen werden automatisch in Teilbeträgen zu festgelegten Zeiten beglichen. Spieler können ihre offenen Rechnungen einsehen und Restbeträge manuell begleichen.
- **Rechnungsverwaltung**: Ein integriertes Menü ermöglicht es Spielern, ihre offenen und bezahlten Rechnungen zu verwalten.
- **Benachrichtigungen**: Automatische Erinnerungen informieren Spieler über ausstehende Zahlungen.
- **Kompatibilität**: Das System ist vollständig kompatibel mit bestehenden ESX-Skripten und erfordert keine Änderungen an anderen Ressourcen.
- **Discord-Integration**: Optionale Webhooks ermöglichen Benachrichtigungen über ausgestellte Rechnungen in Discord-Kanälen.

## myBillingUI – Erweiterung für myBilling

Das **myBillingUI** ist eine optionale Erweiterung, die das Benutzererlebnis durch ein verbessertes Interface optimiert. es ändert nichts an den grundlegenden funktionen von myBilling

### Zusätzliche Funktionen von myBillingUI

- **Visuelle Bestätigung**: Bei Erhalt einer Rechnung erscheint ein Dialogfenster mit der Möglichkeit, die Rechnung zu akzeptieren oder abzulehnen.
- **Benutzerfreundliches Design**: Ein ansprechendes und intuitives Interface verbessert die Interaktion der Spieler mit dem Abrechnungssystem.

### Installation von myBilling

Diese Anleitung beschreibt die Installation und Ersteinrichtung des **myBilling**-Skripts auf deinem FiveM-Server.

#### 1. Voraussetzungen

1. **NativeUI**: Lade **NativeUIReloaded** aus dem **#downloads**-Kanal im Discord herunter, falls es noch nicht installiert ist.
   - Platziere **NativeUIReloaded** im `resources`-Ordner deines Servers.
   - **Hinweis**: Stelle sicher, dass **NativeUIReloaded** vor **myBilling** in der `server.cfg`-Datei geladen wird.

#### 2. Script-Start

1. **Download von myBilling**: Lade das **myBilling**-Skript herunter und entpacke es.
2. **Platzierung**: Verschiebe den Ordner in den `resources`-Ordner deines FiveM-Servers.
3. **Skript aktivieren**: Öffne die `server.cfg`-Datei und füge die folgenden Zeilen hinzu, um **NativeUIReloaded** und **myBilling** zu starten:

    ```lua
    ensure NativeUIReloaded
    ensure myBilling
    ```

4. **Server-Neustart**: Starte deinen Server neu, um die Skripte zu laden.

#### 3. Konfiguration

1. **`config.lua` anpassen**: Navigiere zur `config.lua`-Datei im myBilling-Ordner, um die Einstellungen an deinen Server anzupassen. Hier sind die wichtigsten Konfigurationsoptionen:

    - **`CanEverybodyGiveBills`**: Legt fest, ob alle Spieler Rechnungen ausstellen können (`true` oder `false`).
    - **`WhitelistedJobs`**: Falls nur bestimmte Jobs Rechnungen ausstellen dürfen, gib hier eine Liste der erlaubten Jobs an (z.B. `{'police', 'ambulance'}`).
    - **`PayBillsTime`**: Uhrzeit für automatische Teilzahlungen (im 24-Stunden-Format, z.B. `{h = 21, m = 00}`).
    - **`enableKeyUsage`**: Aktiviert die Möglichkeit, das Rechnungsmenü per Taste zu öffnen (`true` oder `false`).
    - **`Key`**: Wähle die Tasten-ID, um das Rechnungsmenü zu öffnen (z.B. `168` für F7).
    - **`useDiscordWebhooks`**: Aktiviert Webhooks für Discord-Benachrichtigungen (`true` oder `false`).
    - **`Webhooks`**: Setze in der `server.cfg` die gewünschten Webhook-URLs für Benachrichtigungen:

      ```lua
      set webhook_example "https://discordapp.com/api/webhooks/..."
      ```

    Eine vollständige Beschreibung der Konfigurationsoptionen findest du in der [Konfigurationsübersicht](../Konfiguration/).
